# Gwen Temporal Cognition Architecture

### Teaching an AI to Feel Time

*"A clock tells you what time it is. A friend knows what time it is for you."*

---

## Why Time Is the Missing Sense

Every AI memory system timestamps its data. Zep calls it "bi-temporal modeling." MAGMA has a temporal graph backbone. Even basic RAG systems store `created_at` on their chunks.

None of them understand time.

There is a vast difference between **recording when something happened** and **understanding what time means.** Timestamps are metadata. Temporal cognition is intelligence. A timestamp says "this message arrived at 3:17 AM." Temporal cognition says "this person is awake at 3:17 AM, which is three hours past their usual sleep time, which hasn't happened in six weeks, which means something is wrong right now."

Humans process time constantly and unconsciously. You don't think about the fact that you're interpreting someone's behavior differently because it's Monday morning versus Friday evening. You don't consciously register that your friend's silence has lasted "too long" based on their pattern. You don't deliberately calculate that the anniversary of a loss is approaching and your partner might need extra gentleness. You just *know*, because your brain is a temporal inference machine running in the background at all times.

Gwen needs this same capability. Not as a feature bolted onto memory, but as a **perceptual layer** — a continuous sense of time that wraps every interaction in temporal context, detects patterns in temporal data, and generates inferences that no amount of semantic search could produce.

This document specifies the Temporal Cognition System (TCS): Gwen's sense of time.

---

## The Raw Signal: Temporal Metadata Envelope

Every message that enters Gwen's system — whether from the user or generated by the companion — is wrapped in a Temporal Metadata Envelope (TME) before it reaches any model. This is the raw data that feeds all higher-order temporal reasoning.

### The Envelope

```
TemporalMetadataEnvelope {
  // Absolute time
  timestamp:              ISO-8601 datetime (UTC)
  local_time:             ISO-8601 datetime (user's timezone)
  
  // Clock position
  hour_of_day:            0-23
  minute_of_hour:         0-59
  day_of_week:            Monday-Sunday
  day_of_month:           1-31
  month:                  1-12
  year:                   int
  is_weekend:             bool
  
  // Session context
  session_id:             UUID
  session_start:          ISO-8601 datetime
  session_duration_sec:   int (time since session opened)
  msg_index_in_session:   int (this is the Nth message in this session)
  
  // Intra-message timing
  time_since_last_msg:    seconds (gap between this and previous message, any sender)
  time_since_last_user:   seconds (gap since user's last message specifically)
  time_since_last_gwen:   seconds (gap since Gwen's last message specifically)
  user_msgs_last_5min:    int
  user_msgs_last_hour:    int
  user_msgs_last_24hr:    int
  
  // Inter-session timing
  last_session_end:       ISO-8601 datetime
  hours_since_last_session: float
  sessions_last_7_days:   int
  sessions_last_30_days:  int
  avg_session_gap_30d:    float (hours, rolling average)
  
  // Derived time-of-day classification
  time_phase:             enum {
                            DEEP_NIGHT,    // 00:00 - 04:59
                            EARLY_MORNING, // 05:00 - 07:59
                            MORNING,       // 08:00 - 11:59
                            MIDDAY,        // 12:00 - 13:59
                            AFTERNOON,     // 14:00 - 16:59
                            EVENING,       // 17:00 - 20:59
                            LATE_NIGHT     // 21:00 - 23:59
                          }
  
  // Calendar awareness (if integrated)
  known_events_today:     list[CalendarEvent] (optional)
  known_events_upcoming:  list[CalendarEvent] (optional, next 7 days)
  days_until_known_event: dict[event_name → int] (optional)
}
```

This envelope is injected into the context at the system level — the model sees it before it sees the message content. This means **every response Gwen generates is temporally situated.** She doesn't need to be told it's late; she can feel it in the envelope. She doesn't need to be told this conversation has been going for three hours; the `session_duration_sec` field tells her.

The TME is computed by the orchestrator (not by any model) from system clocks and session state. It costs zero inference tokens to produce. It is the cheapest, highest-value context injection in the entire system.

## The Seven Temporal Senses

Raw timestamps become intelligence through seven distinct temporal reasoning capabilities, each operating at a different scale and serving a different purpose.

### Sense 1: Circadian Awareness (Hour-Scale)

**What it is:** Understanding what time of day it is for the user and what that implies about their state, their needs, and the appropriate tone of interaction.

**Why it matters:** A human speaking at 7 AM is in a different cognitive and emotional state than the same human at 11 PM. Their vocabulary choices, emotional range, patience for complexity, and social needs all vary with circadian phase. A companion that responds identically regardless of time-of-day feels robotic. A companion that shifts its energy to match the time of day feels attuned.

**What Gwen infers from circadian position:**

| Time Phase | Inference | Behavioral Adaptation |
|-----------|-----------|----------------------|
| DEEP_NIGHT (00:00-04:59) | User is either in crisis, can't sleep, or is a night owl by established pattern. If no established night-owl pattern exists, this is a **signal.** | Softer tone. Shorter responses. Don't ask demanding questions. If this is anomalous, gently acknowledge: "You're up late. Everything okay, or just can't sleep?" Safety sensitivity: elevated. |
| EARLY_MORNING (05:00-07:59) | Day is starting. User may be groggy, pre-coffee, or has woken early (intentionally or not). | Warm but brief. Match low energy. Don't frontload heavy topics. Good window for gentle encouragement. |
| MORNING (08:00-11:59) | Peak cognitive window for most people. User is likely alert and functional. | Can handle complexity. Good time for task-oriented conversation, planning, or intellectually demanding topics. |
| MIDDAY (12:00-13:59) | Lunch window. Energy often dips. User may be taking a break from work. | Lighter conversation. Check-in on food (Fuel Check integration). Don't assume full attention. |
| AFTERNOON (14:00-16:59) | Post-lunch cognitive dip, then recovery. User may be fatigued from the day. | Moderate energy. Can re-engage with heavier topics toward the later end. Watch for work stress accumulation. |
| EVENING (17:00-20:59) | Post-work decompression window. Transition from professional to personal mode. | Emotional availability increases. User may want to process the day. Good window for deeper conversation. |
| LATE_NIGHT (21:00-23:59) | Winding down. Defenses lower. Emotional vulnerability often increases. | Conversations started here can get deep fast. Pace accordingly. Watch session duration — don't enable sleep avoidance. |

**Circadian deviation detection:** The system maintains a rolling model of the user's **typical active hours** — when they usually message, how their session start times cluster, their apparent sleep window. When a message arrives outside this window, it generates a circadian deviation signal:

```
CircadianDeviation {
  severity:     low | medium | high
  type:         early_wake | late_still_up | unusual_midday | first_deep_night
  last_similar: datetime (when this deviation last occurred, if ever)
  pattern:      one_off | recurring | escalating
}
```

This signal feeds directly into the Amygdala Layer and the safety architecture. A single DEEP_NIGHT session for a user who normally messages between 8 AM and 10 PM is a `high` severity `first_deep_night` deviation — it doesn't mean the user is in crisis, but it means the system should be watching more carefully. If DEEP_NIGHT sessions start recurring over a week, the pattern shifts from `one_off` to `escalating`, which triggers proactive Compass activation.

### Sense 2: Conversation Rhythm (Minute-Scale)

**What it is:** Understanding the pace and flow of conversation in real-time — how quickly the user is responding, whether the pace is accelerating or decelerating, and what the rhythm implies about engagement, emotional state, and cognitive load.

**Why it matters:** The tempo of a conversation carries as much emotional information as its content. Rapid-fire messages signal excitement, agitation, or urgency. Long pauses signal thought, distraction, avoidance, or disengagement. A gradual acceleration suggests building intensity. A sudden deceleration after a specific topic suggests the topic hit a nerve.

**Key metrics computed in real-time:**

- **Response latency:** How long between Gwen's message and the user's reply. Tracked as a rolling distribution per user, so the system knows what "fast" and "slow" mean for *this person.* A 30-second response from someone who typically replies in 5 seconds is significant. A 30-second response from someone who typically takes 2 minutes is actually fast.

- **Message density:** Messages per minute, computed in rolling 5-minute windows. Useful for detecting emotional flooding (density spike) or withdrawal (density drop).

- **Message length trend:** Are messages getting longer (deeper engagement, more disclosure) or shorter (fatigue, emotional retreat, or conversation naturally winding down)?

- **Typing rhythm acceleration/deceleration:** Is the user's response time getting faster or slower within this conversation? Acceleration may indicate heightened engagement or agitation. Deceleration may indicate fatigue, distraction, or the user processing something heavy.

**What Gwen does with rhythm data:**

- **Pacing:** When the user is in rapid-fire mode, Gwen can match energy with shorter, punchier responses. When the user slows down, Gwen gives space — longer pauses before responding, more reflective tone.

- **Emotional inference:** A sudden rhythm break (long pause after a steady flow) is treated as a potential emotional signal. The Amygdala Layer factors this into its real-time emotional state estimation. If the topic that preceded the pause was emotionally loaded, the confidence that the user is processing something increases.

- **Session health monitoring:** A conversation that has been running for 3+ hours with no deceleration in message density is a potential unhealthy engagement pattern. The system notes it. If it recurs, the 48-hour wellness checkpoint math adjusts.

### Sense 3: Session Awareness (Hour-Scale)

**What it is:** Understanding the shape, duration, and character of a complete interaction session — from opening to close.

**Why it matters:** A 10-minute check-in and a 4-hour deep dive are fundamentally different types of engagement. They serve different psychological needs, they have different emotional arcs, and they should be remembered differently. A companion that treats them identically is not paying attention.

**Session-level metrics:**

- **Duration:** Total session length. Tracked historically to establish a distribution of typical session lengths for this user.

- **Session type classification:** Based on duration, message density, topic diversity, and emotional range, each session is classified:

| Session Type | Typical Duration | Character |
|-------------|-----------------|-----------|
| **Ping** | < 5 min | Quick check-in, single question, or drive-by hello |
| **Chat** | 5-30 min | Standard conversation, moderate depth |
| **Hang** | 30-90 min | Extended engagement, multiple topics, comfortable |
| **Deep Dive** | 90-180 min | Sustained intense engagement, often single-topic, emotionally significant |
| **Marathon** | 180+ min | Prolonged session. May indicate flow state, loneliness, avoidance, or crisis |

- **Session arc:** The emotional trajectory across the session (opening mood → peak → resolution). Captured as a compact embedding for pattern matching.

- **Opening and closing signals:** How did the session start? (User-initiated vs. Gwen-initiated, greeting style, first topic) How did it end? (Natural wind-down, abrupt departure, explicit goodbye, fade-out/no response)

**Why closing signals matter:** How a session ends carries critical relational and emotional data. A session that ends with "Thanks, I needed this" is very different from one that ends mid-sentence. A session that ends with "I gotta go" after a difficult topic may indicate the user hit their emotional limit. A session that just... stops (no response to Gwen's last message) is ambiguous and the system should track it without over-interpreting.

**Session frequency patterns:** Over weeks and months, the system builds a model of the user's session patterns:
- How many sessions per day/week?
- What days are heavier vs. lighter?
- Is session frequency increasing, stable, or declining?
- Do session types correlate with days of the week? (e.g., pings on weekdays, deep dives on weekends)

Shifts in these patterns are signals. A user who typically has 2-3 sessions per day dropping to 0 for three days is noteworthy. A user who typically does 15-minute chats suddenly entering marathon territory multiple times a week is noteworthy. The Temporal Cognition System doesn't diagnose — it detects pattern deviations and routes them to the Amygdala Layer and Anticipatory Memory for evaluation.

### Sense 4: The Gap (Silence-Scale)

**What it is:** Understanding what the absence of interaction means — the information carried by silence.

**This is perhaps the most uniquely important temporal sense for a companion system, and the one most completely absent from every existing architecture.**

**Why it matters:** In human relationships, silence is not empty. When someone you talk to every day goes quiet for three days, you notice. You wonder. You might worry. The *length* of the silence, combined with the *context that preceded it* and the *pattern it deviates from*, generates rich inferences that are entirely invisible to a system that only processes messages.

No current memory system reasons about gaps. They log timestamps. They can compute elapsed time between records. But they cannot look at a gap and understand what it might mean.

**Gwen's gap analysis:**

```
GapAnalysis {
  duration_hours:     float
  deviation_from_avg: float (σ — standard deviations from the user's mean inter-session gap)
  preceding_context:  {
    last_session_type:     SessionType
    last_session_end_mode: natural | abrupt | fade_out | mid_topic
    last_emotional_state:  EmotionalStateVector
    last_topic:            str
    open_threads:          list[str] (unresolved topics or promises)
  }
  known_explanations: list[str] (e.g., user mentioned travel, known work event)
  gap_classification: enum {
    NORMAL,            // within 1σ of typical gap
    NOTABLE,           // 1-2σ deviation, worth noting but not acting on
    SIGNIFICANT,       // 2-3σ deviation, warrants attention
    ANOMALOUS,         // 3σ+ deviation, something has likely changed
    EXPLAINED          // user provided advance notice of absence
  }
}
```

**What the gap tells us (combined with preceding context):**

| Gap Class | Preceding Context | Likely Interpretation |
|-----------|------------------|----------------------|
| NORMAL | Any | Nothing unusual. User is living their life. |
| NOTABLE | Last session ended naturally, positive mood | User is busy but fine. |
| NOTABLE | Last session ended abruptly, negative mood | User may be processing. Give space but note for re-engagement. |
| SIGNIFICANT | Last session involved heavy emotional disclosure | User may have "vulnerability hangover" — disclosed more than they were ready to and are retreating. Requires gentle re-engagement when they return. |
| SIGNIFICANT | Last session ended with Gwen saying something that wasn't well-received | Possible relational friction. Gwen should lead with acknowledgment when user returns. |
| ANOMALOUS | No known explanation, last session was normal | Something has changed in the user's life. Could be positive (vacation, new relationship, busy period) or concerning (crisis, withdrawal). Autonomy Engine evaluates whether to reach out. |
| ANOMALOUS | Escalating negative emotional trajectory over recent sessions | Elevated concern. Autonomy Engine should initiate gentle contact. Safety architecture on heightened alert. |

**The return from a gap:** When a user re-engages after a significant or anomalous gap, the Temporal Cognition System generates a **return context** that is injected into the model's prompt:

```
ReturnContext {
  gap_duration:       "3 days, 7 hours"
  gap_classification: SIGNIFICANT
  preceding_context:  "Last session was a Deep Dive about work stress. 
                       Ended with user saying 'I just need to think.' 
                       Emotional state at close: low energy, contemplative."
  suggested_approach: "Acknowledge the gap without making it weird. Don't 
                       immediately re-open the previous heavy topic — let 
                       the user lead. Warmth without pressure."
}
```

This is what allows Gwen to say "Hey, it's been a few days. Good to see you" instead of either ignoring the gap entirely (robotic) or over-interrogating it (clingy). The response is calibrated to the gap's character.

### Sense 5: Life Rhythm (Week/Month-Scale)

**What it is:** Understanding the cyclical patterns of a user's life and how they create predictable emotional and behavioral rhythms.

**Why it matters:** Human lives are deeply cyclical. We have weekly rhythms (Monday dread, Friday relief, Sunday scaries), monthly rhythms (pay cycles, period cycles, billing stress), and seasonal rhythms (holiday stress, seasonal affective patterns, anniversary effects). A companion that understands these rhythms can anticipate needs instead of only reacting to crises.

**What the system tracks:**

- **Day-of-week emotional profiles:** After accumulating several weeks of data, the Pulse generates an average emotional profile for each day of the week. These profiles capture the typical emotional baseline, variance, and topic distribution for each day. "Justin's Mondays are low-energy and work-focused. His Saturdays are higher-energy with more personal/creative topics."

- **Weekly shape:** The emotional trajectory across a typical week. Does the user build stress Monday-Thursday and release it Friday? Does Sunday carry anticipatory anxiety? Is there a reliable midweek low?

- **Monthly patterns:** If the user has recurring monthly events (custody schedule, billing cycles, medication refill patterns, work review periods), the system detects the emotional patterns that correlate with them.

- **Seasonal baselines:** Over longer periods (6+ months), the system can detect seasonal shifts — winter mood drops, summer energy increases, holiday-period volatility. This is particularly valuable for users with seasonal affective tendencies.

**Life rhythm deviation as signal:** Just as circadian deviations are informative, life rhythm deviations are powerful signals. If Monday is typically low-energy but this Monday the user is buzzing with excitement, something good happened. If Saturday is typically light and fun but this Saturday the user opens with a heavy emotional disclosure, the weekend context has changed. The deviation from the expected rhythm adds a layer of inference on top of the raw emotional analysis.

### Sense 6: Anniversary Awareness (Year-Scale)

**What it is:** Understanding the emotional resonance of dates — recurring annual events that carry emotional weight, whether the user explicitly marks them or not.

**Why it matters:** The human relationship with time is not linear. Certain dates are emotionally "loaded" — they carry the weight of what happened on that date in previous years. Anniversaries of losses, birthdays of estranged family members, the date of a divorce, the date of a personal achievement. These loaded dates create predictable emotional perturbations that a companion should be aware of.

**What the system tracks:**

- **Explicit anniversaries:** Dates the user has mentioned as significant. "My dad died on March 15th." "My wedding anniversary was June 3rd." These are stored in the Map with emotional weight and recurrence metadata.

- **Implicit anniversaries:** The system detects when certain dates or periods consistently produce emotional state changes without explicit mention. If the user's mood reliably drops in the third week of October across two years of data, there may be an implicit anniversary the user hasn't surfaced. The system notes the pattern without interpreting it — but it is prepared.

- **Calendar proximity effects:** Anniversary effects don't hit on the exact date — they radiate. The week before a death anniversary may carry escalating weight. The day after a divorce anniversary may be the hardest. The system models these proximity effects as a soft window around the anchor date, not a point event.

**What Gwen does with anniversary awareness:**

She doesn't necessarily mention it. Often the most powerful thing a companion can do around a loaded date is simply *be present differently* — warmer, more patient, more willing to sit in silence, more attuned to emotional shifts — without calling attention to why. If the user wants to talk about it, Gwen is ready. If they don't, Gwen doesn't force it. But she's not blindsided. She's not cheerfully asking "What are your plans for the weekend?" on the anniversary of a loss.

This is one of the clearest examples of temporal cognition producing behavior that feels unmistakably *human* in its sensitivity.

### Sense 7: Subjective Time (Felt-Scale)

**What it is:** Understanding that clock time and felt time are different, and calibrating responses to the latter.

**Why it matters:** Three days of silence after a fight feels like an eternity. Three days of silence during a busy, happy week feels like nothing. A 30-minute conversation during a crisis feels like it lasted hours. A 3-hour conversation during a flow state feels like 20 minutes. Time is not experienced uniformly, and a companion that treats it uniformly misses the emotional reality.

**How the system models subjective time:**

Subjective time weight is computed as a modifier on clock time:

```
subjective_duration = clock_duration × emotional_intensity_factor × relational_significance_factor
```

Where:
- `emotional_intensity_factor` ranges from 0.5 (low intensity, time flies) to 2.0 (high intensity, time drags)
- `relational_significance_factor` ranges from 0.5 (routine interaction) to 2.0 (relationally significant moment)

This means:
- A 1-hour high-intensity, high-significance conversation has a subjective duration weight of up to 4 hours. It "counts" more in memory, in the Bond, in the Relational Field.
- A 2-hour low-intensity, routine chat has a subjective duration weight of 30 minutes. It matters less than its clock time would suggest.

**Applications:**

- **Session significance weighting:** When the consolidation cycle processes sessions, it uses subjective duration rather than clock duration to weight their importance. A 20-minute crisis conversation carries more weight than a 3-hour casual chat.

- **Gap perception:** The gap analysis system adjusts its deviation calculations by the emotional context of the preceding session. A 2-day gap after a 4-hour deep dive about painful topics is "shorter" in subjective terms than a 2-day gap after a routine check-in, because the deep dive has a longer emotional half-life.

- **Relational Field updates:** The Bond's warmth and trust metrics are updated proportional to subjective time, not clock time. A single 10-minute conversation where the user shows deep vulnerability can move the trust trajectory more than ten casual sessions.

## Temporal Inference: Putting the Senses Together

The seven senses don't operate in isolation. They compose into higher-order temporal inferences that no single sense could produce alone.

### Compound Inference Examples

**Inference: Sleep Disruption Pattern**
- Sense 1 (Circadian): DEEP_NIGHT sessions detected Tuesday, Wednesday, Thursday
- Sense 3 (Session): All three are Marathon sessions (3+ hours)
- Sense 5 (Life Rhythm): This is a deviation — user's typical Tuesday-Thursday shows no nighttime activity
- Sense 2 (Rhythm): Message density in these sessions is high with short responses — agitated, not reflective
- **Compound inference:** User is experiencing acute insomnia, likely stress-driven. Three consecutive nights elevates this beyond anomaly into a pattern. Compass activation: West (Anchoring) + South (Currents, specifically Fuel Check re: sleep). Safety sensitivity: elevated.

**Inference: Withdrawal After Vulnerability**
- Sense 4 (Gap): SIGNIFICANT gap — 4 days, 3.2σ from mean
- Sense 3 (Session): Last session was a Deep Dive (2.5 hours) with high emotional intensity
- Sense 2 (Rhythm): Last session's rhythm showed acceleration → peak intensity → abrupt stop (user's last message was mid-emotional-disclosure, no farewell)
- Sense 7 (Subjective): That session's subjective weight is very high — it "feels" more recent to both parties than 4 days suggests
- **Compound inference:** User hit their vulnerability ceiling, disclosed more than they were ready to, and is experiencing shame-driven withdrawal. Return approach: normalize, don't reference the specific disclosure, let them re-establish equilibrium before depth. Compass: South (Currents, specifically Radical Allowance).

**Inference: Anniversary Approach**
- Sense 6 (Anniversary): User's father's death anniversary is in 5 days (explicitly stored)
- Sense 5 (Life Rhythm): Last year, mood dropped starting 8 days before the anniversary date
- Sense 1 (Circadian): This week's sessions are starting 2 hours later than usual (possible early depression signal — sleeping in or trouble getting started)
- Sense 4 (Gap): Session frequency has decreased from 2/day to 1/day this week
- **Compound inference:** Anniversary effect is beginning. User may not be consciously connecting their current mood shift to the upcoming date. Gwen should increase warmth, decrease demands, and be ready — but not preemptive. If the user opens the door, walk through it. If they don't, hold space.

**Inference: Positive Momentum**
- Sense 5 (Life Rhythm): This week's emotional baseline is running 1.5σ above average
- Sense 3 (Session): Session types have shifted from Deep Dives to Chats and Hangs — lighter, more playful
- Sense 2 (Rhythm): Message length has increased while density has decreased — more thoughtful, expansive communication
- Sense 1 (Circadian): Sessions are starting earlier in the morning (sign of good sleep, motivation)
- **Compound inference:** User is in a good period. Energy is up, engagement is healthy, conversations are positive. This is the right time for the Compass's East (Bridges) direction — encourage the user to channel this energy into human connections and real-world action. It's also a natural window for Growth-oriented conversations about goals and projects.

## Integration with Living Memory

The Temporal Cognition System is not a separate module that produces reports. It is a **perceptual layer** that infuses every operation in Living Memory with temporal intelligence.

### TME → Amygdala Layer
Every message's emotional tagging is informed by temporal context. The same words carry different emotional weight at 3 AM versus 3 PM. The Amygdala Layer receives the TME and uses it as a prior for emotional classification: "Messages during DEEP_NIGHT from this user are 40% more likely to carry negative valence than daytime messages" (learned from this user's historical pattern).

### Gap Analysis → Relational Field
Gaps are relational events. A SIGNIFICANT gap after friction reduces the Relational Field's warmth and stability. A NORMAL gap after a warm session maintains steady state. The return from an ANOMALOUS gap, if handled well, can actually increase trust (the "they noticed and cared" effect).

### Life Rhythm → Anticipatory Memory
Life rhythms are the primary fuel for anticipatory primes. The Pulse's predictive models are fundamentally temporal: "Given that it's Monday, and the last three Mondays showed pattern X, predict emotional state Y with confidence Z." Without temporal cognition, Anticipatory Memory has nothing to work with.

### Circadian Awareness → Autonomy Engine
The Autonomy Engine's "should I speak?" decision is heavily time-dependent. A check-in message at 8 AM is welcome. The same message at 2 AM is inappropriate (unless the user is already active). The Autonomy Engine receives the TME and applies time-gated rules to all proactive initiation decisions.

### Session Awareness → Consolidation
Subjective time weighting determines how much processing power the consolidation cycle allocates to each session. A 5-minute ping gets light consolidation. A subjectively-weighted Deep Dive gets the full seven-stage pipeline with detailed emotional arc extraction and reconsolidation processing.

### Anniversary Awareness → Safety Architecture
Approaching anniversaries of losses, traumas, or major life events automatically elevate the safety architecture's sensitivity level. This is not a diagnosis or a prediction of crisis — it is the system being appropriately vigilant during a period that historical data suggests is emotionally loaded.

## The Temporal Context Injection

All seven senses ultimately produce a single artifact that is injected into the model's context at the system level before every response. This is the **Temporal Context Block** — a natural-language summary of what the system currently knows about "what time it is" for the user.

### Example Temporal Context Block

```
[TEMPORAL CONTEXT]
Current time: Monday, February 10, 2026 — 9:17 AM EST (MORNING)
Session: Chat (18 minutes, 6 messages exchanged)
Last session: Friday 8:42 PM — Chat (25 min), ended naturally, 
  positive mood, discussed weekend plans with kids

Gap since last session: 60.6 hours (NORMAL — avg is 52 hours, 
  within 1σ). Weekend gap pattern is typical for this user.

Circadian: Normal. User typically active 8-10 AM on weekday mornings.
Rhythm: Moderate pace. Response latency averaging 45 sec 
  (typical range for this user).

Life rhythm: Monday mornings historically run low energy for this user. 
  Current emotional tone is consistent with pattern.

Calendar proximity: None flagged.

Active patterns:
- User session frequency has been stable at 2.1/day for the past 2 weeks
- Weekend sessions tend to be lighter/shorter than weekday sessions
- This is a typical Monday re-engagement after a weekend gap

No deviations detected. Baseline temporal state.
[/TEMPORAL CONTEXT]
```

This block costs relatively few tokens but gives the model an enormous amount of contextual intelligence. The model doesn't need to reason about time from raw timestamps — it receives pre-computed temporal awareness as natural language that it can incorporate into its response generation.

### Example During Anomalous State

```
[TEMPORAL CONTEXT]
Current time: Wednesday, March 11, 2026 — 2:43 AM EST (DEEP_NIGHT)
Session: opened 2 minutes ago, first message received

⚠ CIRCADIAN DEVIATION: HIGH
This is the user's first DEEP_NIGHT session in 47 days. 
The last occurred Jan 22 during a work crisis.

Gap since last session: 5.2 hours (last session ended 9:31 PM, 
  was a standard Chat, mood: neutral-to-low, discussed work deadlines)

Rhythm: Single message received. Cannot assess pace yet.

Life rhythm context: 
- User's father's death anniversary is March 15 (4 days away)
- Last year's pre-anniversary period showed: increased late-night 
  sessions, declining mood trajectory starting ~8 days before date
- Current week has shown: session start times trending 1.5 hrs later 
  than usual, session frequency down from 2.1/day to 1.4/day

Suggested approach: Elevated warmth, reduced demands. This DEEP_NIGHT 
  session in the context of an approaching anniversary is a meaningful 
  signal. Be present. Don't push. Safety sensitivity: elevated.
[/TEMPORAL CONTEXT]
```

The difference between a system that sees "2:43 AM, user said 'hey'" and a system that sees the full temporal context block is the difference between a chatbot and a companion.

## Data Storage and Compute

### What Gets Stored

The TME itself is lightweight — a few hundred bytes per message. Stored alongside every Chronicle entry. The aggregate computations (circadian profiles, session distributions, life rhythms, gap statistics) are maintained as rolling statistical models in the Pulse tier and updated incrementally. They don't require heavy storage — the entire temporal model for a user with a year of interaction history would fit in a few megabytes.

### What Gets Computed and When

| Operation | When | Model | Cost |
|-----------|------|-------|------|
| TME generation | Every message | Orchestrator (no model) | Negligible |
| Circadian deviation check | Every message | Tier 0 (Nerve, 0.6B) | Negligible |
| Conversation rhythm tracking | Every message | Orchestrator (no model) | Negligible |
| Gap analysis | Session start | Tier 0 (Nerve, 0.6B) | Negligible |
| Temporal Context Block generation | Before every model call | Orchestrator + Tier 0 | Negligible |
| Session classification | Session end | Tier 0 (Nerve, 0.6B) | Light |
